Open Data for Open Lands

The federal government has released a draft contract that will define how we access our public lands for the next decade. They are seeking a private contractor to build software that will provide online access to our nation’s parks, forests, monuments, campsites, cabins, and tours. As drafted, this contract places all the control into the hands of one contractor and one website, creating a closed silo and a monopoly.

We have successfully pushed to have the comment period on this draft extended until Nov. 7th, so we have the attention of Whitehouse officials and Congress. Now, we need your help in sending the message that this is a big step backwards for the Obama Adminstration and the contract needs to include the creation of an API. An API would allow various applications to share the data and improve everyone’s access to public lands.

What to say

Example 1: President Obama has made it clear that he wants to “make sure that we’re giving entrepreneurs the ability, if we build an effective platform, to essentially develop apps that work off this new information.” To follow through with that promise, we must insist on the creation of an API for public land data. Anything less than that would be a big step back and out of step with the administration’s open data policy.
Example 2: This case is specifically about public land, but it’s representative of the larger movement towards the need to create platforms to access public data instead of silos. Silos create monopolies and a lack of innovation, whereas platforms create healthy competition, more iteration and, in turn, better end products. Opening up this system to an API is an important step in the creation of a platform to ensure the U.S. keeps up with global innovation best practices.
